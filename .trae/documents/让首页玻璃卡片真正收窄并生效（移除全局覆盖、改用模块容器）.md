## 原因定位
- 首页仍然太宽，说明我们给 `.glassPanel` 的收窄样式被全局样式（如 `.home-page/.breathing-section`）覆盖。当前 `Home.jsx` 还有这两个全局类，导致模块化样式不生效或被覆盖。

## 调整方案
1. 移除首页的全局容器类：把 `Home.jsx` 外层 `className="home-page"` 与 `className="breathing-section"` 改为模块类 `className={styles.page}` 与 `className={styles.container}`，彻底使用 CSS Modules，避免全局覆盖。
2. 明确卡片宽度与居中：在 `Home.module.css` 设置
   - `.glassPanel { width: 100%; max-width: clamp(460px, 70vw, 600px); margin: 60px auto; padding: 28px 24px; border-radius: 24px; }`
   - 保留液态玻璃风格（边框、模糊、渐变、阴影）。
3. 收窄时间选择条并居中：
   - `.controls { display:flex; flex-direction:column; align-items:center; gap:10px }`
   - `.glassPanel input[type="range"], .progress { width: 60% }`
   - 移动端（≤640px）将两者调为 `85%`。
4. 标题与间距优化：
   - `.title { font-size: clamp(1.6rem, 4vw, 2.2rem); margin-bottom: 8px; }`
5. 验证与清理：
   - 检查 `App.css` 或其他全局样式中是否仍有 `.breathing-section/.home-page` 定义（如宽度、布局），若有则仅首页不使用这些类，避免影响其他页面。

## 预期效果
- 首页玻璃卡片真正变窄（约 460–600px，自适应居中），布局与示例一致：标题 → 呼吸球 → 更短滑块 → 按钮。
- 风格统一为液态玻璃，移动端自适应良好。
