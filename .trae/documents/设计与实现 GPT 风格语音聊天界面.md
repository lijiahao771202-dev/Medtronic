## 信息架构
1. 顶部栏（TopBar）
- 左：品牌与当前会话状态（圆点+文案：已连接/正在聆听/正在回应/语音错误）。
- 右：设置图标与“提示词”按钮（打开编辑弹层，默认不展示内容）。

2. 中心互动区（Center Orb）
- 大型圆形“语音球”带液态玻璃视觉：
  - 动态折射/扭曲：SVG 滤镜（feTurbulence + feDisplacementMap），随鼠标/触控位置轻微变形。
  - 交互弹性：点击/长按触发弹性缩放（弹簧动画）。
  - 色散效果：边缘微小 RGB 分离（多层阴影/滤镜，透明度极低）。
- 球体内外环显示实时音量/播放频谱（使用现有 `AnalyserNode`）。

3. 底部控制条（Bottom Controls）
- 主按钮：开始讲话/停止讲话（切换识别）。
- 次按钮：暂停/继续播放（TTS 会话），挂断（结束并返回）。
- 辅助：显示/隐藏文字（转录与生成结果），错误详情入口（代理状态码与摘要）。

4. 文本区（Transcript）
- 可折叠的轻量转录面板（您之前的“显示文字/隐藏文字”），保持极简，不干扰沉浸。

## 视觉与动效设计（Liquid Glass）
- 背景：柔和渐变 + 轻噪声，前景控件采用玻璃态卡片（低透明度 + `backdrop-filter: blur()`）。
- 球体：
  - 形状：圆形容器（CSS Modules 局部样式），叠加 SVG Filter 产生折射位移；
  - 弹性：Framer Motion（或 CSS transition 的 spring 近似）用于缩放/抖动；
  - 色散：多层 `filter: drop-shadow()` 或 `text-shadow` 的伪元素轻微偏移 R/G/B；
- 频谱：环形柱状或放射条（基于 Canvas 2D），颜色主调为蔚蓝系，与您当前 `#5aa0b3` 保持一致。

## 交互状态与事件流
- 识别：点击主按钮 → 恢复 `AudioContext` → 启动 Web Speech → 顶部状态切换至“正在聆听”。静音 1.5s 自动结束一次识别。
- 生成：收到最终识别 → 调用代理 → 顶部状态“正在回应”；异常则在“错误详情”面板展示。
- 播放：严格按原文分段与指令执行（不做规范化）；暂停/继续仅影响当前播放会话，不销毁 `AudioContext`。
- 清理：每轮开始前停止上一轮所有 `BufferSource`，保持顺序与唯一播放，避免叠加。

## 响应式与可访问性

- 平板/桌面：中心球体 420–560px 直径，底部控制横排；转录面板宽度自适应。
- 可访问性：按钮提供 `aria-label`；颜色对比度达标；错误文本可见。
